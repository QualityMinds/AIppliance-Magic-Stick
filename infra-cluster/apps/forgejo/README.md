# Forgejo (production-ready) in namespace `development`

This directory deploys Forgejo through Flux `HelmRelease` into Kubernetes namespace `development`.

## Included resources

- `helmrepository.yaml`: Forgejo Helm chart source
- `helmrelease.yaml`: production-oriented Forgejo deployment configuration
- `secret-admin.yaml`: bootstrap admin credentials (password generated by secret-generator)
- `secret-security.yaml`: Forgejo security keys/tokens (generated)
- `secret-postgresql.yaml`: PostgreSQL password (generated)

## Prerequisites

- Flux is installed and reconciling `deployments/CHANGEME_DEPLOYMENT/infra-cluster/flux-bootstrap`
- `infrastructure-basis/secret-generator` is running
- Ingress controller (`ingress-nginx`) is available
- DNS record for `forgejo.example.local` points to the ingress endpoint
- cert-manager cluster issuer exists for `cert-manager.io/cluster-issuer` annotation

## Required configuration points

Review and adjust these values before production rollout:

1. In `helmrelease.yaml`:
   - `ingress.annotations.cert-manager.io/cluster-issuer`: replace `REPLACE_WITH_CLUSTER_ISSUER`
   - `ingress.hosts[0].host` and `ingress.tls[0].hosts[0]`: set your Forgejo FQDN
   - `gitea.config.server.DOMAIN` and `ROOT_URL`: align to the ingress host
   - `persistence.storageClass` and `postgresql.primary.persistence.storageClass`: set your storage class (or keep default)
2. In `secret-admin.yaml`:
   - `username` and `email` for the initial admin account

## Security and production defaults

- Persistent volumes enabled for Forgejo data and PostgreSQL data
- TLS ingress configured with dedicated TLS secret (`forgejo-tls`)
- Registration disabled by default (`DISABLE_REGISTRATION: true`)
- Secrets separated from non-sensitive config and generated via secret-generator
- Container security hardening defaults (`allowPrivilegeEscalation: false`, dropped Linux capabilities)

## Deploy / reconcile

```bash
kubectl kustomize deployments/CHANGEME_DEPLOYMENT/infra-cluster/flux-bootstrap | kubectl apply -f -
flux reconcile kustomization apps --with-source -n flux-system
```

## Operations

```bash
# Check Flux objects
kubectl -n flux-system get helmrepository,helmrelease | grep forgejo

# Check generated secrets
kubectl -n development get secret forgejo-admin-secret forgejo-security-secret forgejo-postgresql-secret

# Check workload and ingress
kubectl -n development get pods,svc,ingress,pvc

# Check admin bootstrap values (example: username/email)
kubectl -n development get secret forgejo-admin-secret -o jsonpath='{.data.username}' | base64 -d; echo
kubectl -n development get secret forgejo-admin-secret -o jsonpath='{.data.email}' | base64 -d; echo
```
